## 1. 飞书(Feishu)配置

### 1.1 概述

飞书是字节跳动旗下的企业协作平台，OpenClaw通过飞书开放平台API与飞书进行集成。

### 1.2 创建飞书应用

#### 步骤1：访问飞书开放平台

1. 打开 [飞书开放平台](https://open.feishu.cn/)
2. 使用企业管理员账号登录
3. 点击右上角「开发者后台」

#### 步骤2：创建企业自建应用

1. 在开发者后台，点击「创建应用」
2. 选择「企业自建应用」
3. 填写应用信息：
   - 应用名称：例如 "OpenClaw助手"
   - 应用描述：AI助手集成应用
   - 应用图标：上传应用图标（可选）
4. 点击「创建应用」

#### 步骤3：获取App ID和App Secret

1. 进入应用详情页
2. 在左侧菜单选择「凭证与基础信息」
3. 复制以下信息：
   - **App ID**（应用ID）：格式如 `cli_xxxxxxxxxxxx`
   - **App Secret**（应用密钥）：点击「查看」按钮获取

```yaml
# 配置示例
feishu:
  app_id: "cli_xxxxxxxxxxxxxxxx"
  app_secret: "xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
  encrypt_key: ""  # 可选，用于事件加密
  verification_token: ""  # 可选，用于事件验证
```

### 1.3 配置权限

#### 步骤1：添加机器人能力

1. 在应用详情页，点击「添加应用能力」
2. 选择「机器人」
3. 点击「添加」

#### 步骤2：配置权限范围

1. 进入「权限管理」页面
2. 点击「申请权限」，添加以下权限：

| 权限名称 | 权限Key | 说明 |
|---------|---------|------|
| 获取用户ID | `contact:user.id:readonly` | 获取用户身份信息 |
| 获取用户基本信息 | `contact:user.base:readonly` | 读取用户基本信息 |
| 获取部门信息 | `contact:department.base:readonly` | 读取组织架构 |
| 读取消息 | `im:message:readonly` | 接收用户消息 |
| 发送消息 | `im:message:send` | 向用户发送消息 |
| 读取群信息 | `im:chat:readonly` | 获取群聊信息 |
| 读取群成员信息 | `im:chat:members:readonly` | 获取群成员列表 |
| 获取群组信息 | `im:chat.group:readonly` | 读取群组详情 |
| 管理群组 | `im:chat.group:manage` | 管理群组设置 |

3. 点击「申请」提交权限申请
4. 联系企业管理员审批权限

#### 步骤3：配置事件订阅

1. 进入「事件订阅」页面
2. 开启「事件订阅」开关
3. 配置请求地址（Request URL）：
   - 格式：`https://your-domain.com/webhook/feishu`
   - 或：`https://your-domain.com/api/feishu/events`
4. 点击「验证」按钮验证URL可访问性

#### 步骤4：订阅事件类型

添加以下事件订阅：

| 事件类型 | 事件Key | 说明 |
|---------|---------|------|
| 接收消息 | `im.message.receive_v1` | 接收用户发送的消息 |
| 群组被创建 | `im.chat.group.created_v1` | 监听群组创建事件 |
| 用户进入群组 | `im.chat.member.user.added_v1` | 监听用户进群事件 |
| 用户离开群组 | `im.chat.member.user.deleted_v1` | 监听用户退群事件 |

### 1.4 发布应用

1. 进入「版本管理与发布」页面
2. 点击「创建版本」
3. 填写版本信息：
   - 版本号：如 1.0.0
   - 更新说明：初始版本
4. 点击「申请发布」
5. 联系企业管理员审批发布

### 1.6 常见问题排查

#### 问题1：事件订阅验证失败

**现象**：配置Request URL时提示验证失败

**排查步骤**：
1. 确认服务器可公网访问
2. 检查URL是否正确，需包含协议（https://）
3. 确认服务器正确返回challenge参数
4. 检查防火墙是否放行端口

**解决方案**：
```python
# Flask示例 - 正确处理challenge验证
@app.route('/webhook/feishu', methods=['POST'])
def feishu_webhook():
    data = request.json
    
    # 处理URL验证
    if data.get('type') == 'url_verification':
        return jsonify({"challenge": data['challenge']})
    
    # 处理事件
    handle_event(data)
    return jsonify({"code": 0})
```

#### 问题2：权限申请被拒绝

**现象**：权限申请状态显示「被拒绝」

**解决方案**：
1. 确认申请的权限与应用功能匹配
2. 联系企业管理员说明权限用途
3. 检查是否申请了敏感权限（如通讯录全量读取）

#### 问题3：机器人收不到消息

**现象**：用户@机器人但无响应

**排查步骤**：
1. 确认机器人已添加到群组
2. 检查事件订阅中是否包含 `im.message.receive_v1`
3. 确认应用已发布并通过审核
4. 查看事件订阅日志，确认事件是否送达

#### 问题4：Token过期

**现象**：API调用返回 `99991663` 错误码

**解决方案**：
```python
# 实现Token自动刷新
class FeishuClient:
    def __init__(self, app_id, app_secret):
        self.app_id = app_id
        self.app_secret = app_secret
        self._token = None
        self._token_expire = 0
    
    def get_access_token(self):
        if time.time() < self._token_expire - 300:  # 提前5分钟刷新
            return self._token
        
        # 刷新Token
        resp = requests.post(
            "https://open.feishu.cn/open-apis/auth/v3/tenant_access_token/internal",
            json={"app_id": self.app_id, "app_secret": self.app_secret}
        )
        data = resp.json()
        self._token = data['tenant_access_token']
        self._token_expire = time.time() + data['expire']
        return self._token
```

## 2. Telegram配置

### 2.1 概述

Telegram是一款注重隐私和安全的即时通讯应用，通过Bot API与OpenClaw集成。

### 2.2 创建Bot

#### 步骤1：联系BotFather

1. 打开Telegram应用
2. 搜索 `@BotFather` 或访问 [BotFather](https://t.me/botfather)
3. 点击「Start」开始对话

#### 步骤2：创建新Bot

1. 发送命令 `/newbot`
2. 按提示设置Bot信息：
   - **名称**：显示名称，如 "OpenClaw助手"
   - **用户名**：唯一标识，必须以 `bot` 结尾，如 `openclaw_bot`

#### 步骤3：获取Bot Token

创建成功后，BotFather会发送消息：

```

123456789:ABCdefGHIjklMNOpqrSTUvwxyz1234567890

Keep your token secure and store it safely, it can be used by anyone to control your bot.
For a description of the Bot API, see this page: https://core.telegram.org/bots/api
```

**保存Token**：`123456789:ABCdefGHIjklMNOpqrSTUvwxyz1234567890`

### 2.3 配置Webhook

#### 方式1：Webhook模式（推荐用于生产环境）

**步骤1：准备HTTPS服务器**

Webhook需要有效的SSL证书（不能使用自签名证书）。

**步骤2：设置Webhook**

```bash
# 使用curl设置Webhook
curl -X POST "https://api.telegram.org/bot<TOKEN>/setWebhook" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://your-domain.com/webhook/telegram",
    "allowed_updates": ["message", "callback_query", "inline_query"],
    "drop_pending_updates": true
  }'
```

**步骤3：验证Webhook设置**

```bash
# 获取Webhook信息
curl "https://api.telegram.org/bot<TOKEN>/getWebhookInfo"
```

#### 方式2：Long Polling模式（适合开发环境）

```python
# Python示例
import requests
import time

TOKEN = "123456789:ABCdefGHIjklMNOpqrSTUvwxyz1234567890"
BASE_URL = f"https://api.telegram.org/bot{TOKEN}"

def get_updates(offset=None):
    params = {"timeout": 30}
    if offset:
        params["offset"] = offset
    response = requests.get(f"{BASE_URL}/getUpdates", params=params)
    return response.json()

def main():
    offset = None
    while True:
        updates = get_updates(offset)
        if updates.get("result"):
            for update in updates["result"]:
                handle_update(update)
                offset = update["update_id"] + 1
        time.sleep(1)
```

通过BotFather设置命令菜单：

1. 发送 `/setcommands` 给BotFather

3. 发送命令列表：


```python
# Python SDK示例
from telegram import Update, Bot
from telegram.ext import Application, CommandHandler, MessageHandler, filters

TOKEN = "123456789:ABCdefGHIjklMNOpqrSTUvwxyz1234567890"

async def start(update: Update, context):
    await update.message.reply_text("欢迎使用OpenClaw!")

async def handle_message(update: Update, context):
    text = update.message.text
    chat_id = update.message.chat_id
    
    # 处理消息
    response = process_message(text)
    await context.bot.send_message(
        chat_id=chat_id,
        text=response,
        parse_mode="Markdown"
    )

# 创建应用
application = Application.builder().token(TOKEN).build()

# 添加处理器
application.add_handler(CommandHandler("start", start))
application.add_handler(MessageHandler(filters.TEXT & ~filters.COMMAND, handle_message))

# 启动（Webhook模式）
application.run_webhook(
    listen="0.0.0.0",
    port=8443,
    webhook_url="https://your-domain.com/webhook/telegram"
)
```

### 2.6 常见问题排查

#### 问题1：Webhook设置失败

**现象**：调用setWebhook返回错误

**排查步骤**：
1. 确认URL使用HTTPS协议
2. 检查SSL证书是否有效（不能是自签名证书）
3. 确认服务器可公网访问
4. 检查证书链是否完整

**常用命令**：
```bash
# 检查证书有效性
openssl s_client -connect your-domain.com:443 -servername your-domain.com

# 测试Webhook端点
curl -v https://your-domain.com/webhook/telegram
```

#### 问题2：收不到消息

**现象**：用户发送消息但Bot无响应

**排查步骤**：
1. 确认Bot已启动（向Bot发送 `/start`）
2. 检查Webhook URL是否正确
3. 查看服务器日志确认请求是否到达
4. 使用getUpdates检查是否有积压消息

```bash
# 清除Webhook，切换到Polling模式测试
curl "https://api.telegram.org/bot<TOKEN>/deleteWebhook?drop_pending_updates=true"
```

#### 问题3：消息格式错误

**现象**：发送消息返回400错误

**解决方案**：
```python
# 正确处理Markdown转义
def escape_markdown(text):
    escape_chars = ['_', '*', '[', ']', '(', ')', '~', '`', '>', '#', '+', '-', '=', '|', '{', '}', '.', '!']
    for char in escape_chars:
        text = text.replace(char, f'\\{char}')
    return text

# 发送消息时处理异常
try:
    await bot.send_message(chat_id, text, parse_mode="Markdown")
except Exception as e:
    # 如果Markdown解析失败，使用纯文本发送
    await bot.send_message(chat_id, text)
```

#### 问题4：Bot被限制

**现象**：无法向某些用户发送消息

**原因**：
- 用户未与Bot开始对话（未发送 `/start`）
- 用户已屏蔽Bot
- 用户已删除与Bot的聊天

**解决方案**：
```python
# 检查是否可以发送消息
async def can_send_message(bot, chat_id):
    try:
        chat = await bot.get_chat(chat_id)
        return True
    except Exception as e:
        if "bot was blocked" in str(e):
            return False
        return False
```

## 3. Slack配置

### 3.1 概述

Slack是企业协作平台，OpenClaw支持通过Slack API进行集成，包括Socket Mode和HTTP Mode两种方式。

### 3.2 创建Slack应用

#### 步骤1：访问Slack API

1. 打开 [Slack API](https://api.slack.com/)
2. 点击「Create New App」

#### 步骤2：创建应用

选择创建方式：
- **From scratch**：从头创建
- **From an app manifest**：从配置清单创建

选择「From scratch」：
1. 输入应用名称，如 "OpenClaw"
2. 选择工作区（Workspace）
3. 点击「Create App」

### 3.3 配置权限范围

#### 步骤1：添加Bot Token权限

1. 在应用管理页面，选择「OAuth & Permissions」
2. 滚动到「Scopes」部分
3. 在「Bot Token Scopes」中添加以下权限：

| 权限 | 范围 | 说明 |
|------|------|------|
| 查看用户基本信息 | `users:read` | 获取用户信息 |
| 查看频道信息 | `channels:read` | 读取频道列表 |
| 查看群组信息 | `groups:read` | 读取私有频道 |
| 发送消息 | `chat:write` | 发送消息 |
| 发送私信 | `chat:write.public` | 发送到公共频道 |
| 查看消息 | `channels:history` | 读取频道消息历史 |
| 查看IM消息 | `im:history` | 读取私信历史 |
| 查看群组消息 | `groups:history` | 读取群组消息 |
| 查看IM频道 | `im:read` | 读取私信频道 |
| 查看群组频道 | `mpim:read` | 读取多人私信 |
| 查看表情反应 | `reactions:read` | 读取表情反应 |
| 添加表情反应 | `reactions:write` | 添加表情反应 |
| 查看文件 | `files:read` | 读取文件信息 |

#### 步骤2：添加用户Token权限（可选）

如果需要以用户身份操作，添加「User Token Scopes」：

| 权限 | 范围 | 说明 |
|------|------|------|
| 查看用户资料 | `users:read` | 读取用户资料 |
| 查看用户邮箱 | `users:read.email` | 读取用户邮箱 |
| 发送消息 | `chat:write` | 发送消息 |

#### 步骤3：安装应用到工作区

1. 在「OAuth & Permissions」页面
2. 点击「Install to Workspace」
3. 确认权限并授权
4. 复制「Bot User OAuth Token」（以 `xoxb-` 开头）

### 3.4 启用Socket Mode

Socket Mode适合内网部署或无法提供公网URL的场景。

#### 步骤1：启用Socket Mode

1. 在应用管理页面，选择「Socket Mode」
2. 开启「Enable Socket Mode」
3. 生成App-Level Token：
   - 点击「Generate Token and Scopes」
   - 添加权限范围：`connections:write`
   - 复制Token（以 `xapp-` 开头）

#### 步骤2：订阅事件

1. 选择「Event Subscriptions」
2. 开启「Enable Events」
3. 在「Subscribe to bot events」中添加：

| 事件 | 说明 |
|------|------|
| `message.channels` | 监听频道消息 |
| `message.groups` | 监听私有频道消息 |
| `message.im` | 监听私信消息 |
| `message.mpim` | 监听多人私信 |
| `app_mention` | 监听@Bot的消息 |
| `reaction_added` | 监听表情反应 |

4. 点击「Save Changes」

### 3.5 配置HTTP Mode（可选）

如果不使用Socket Mode，可以使用HTTP模式：

1. 在「Event Subscriptions」中配置Request URL：
   - URL格式：`https://your-domain.com/webhook/slack`
2. 启用「Enable Events」
3. 配置相同的Bot事件订阅

### 3.6 配置示例代码

```yaml
# config/slack.yaml
channels:
  slack:
    enabled: true
    
    # 连接模式: socket 或 http
    mode: "socket"
    
    # Token配置
    bot_token: "${SLACK_BOT_TOKEN}"  # xoxb-开头的Bot Token
    app_token: "${SLACK_APP_TOKEN}"  # xapp-开头的App Token（Socket Mode需要）
    signing_secret: "${SLACK_SIGNING_SECRET}"  # HTTP Mode需要

    socket:
      ping_interval: 30
      
    # HTTP Mode配置
    http:
      path: "/webhook/slack"
      port: 8080

    message_handler:
      # 处理@提及
      handle_mentions: true
      # 处理所有消息
      handle_all_messages: false
      # Bot用户ID
      bot_user_id: "Uxxxxxxxx"
```

```python
# Python SDK示例 - Socket Mode
import os
from slack_bolt import App
from slack_bolt.adapter.socket_mode import SocketModeHandler

# 初始化应用
app = App(token=os.environ.get("SLACK_BOT_TOKEN"))

@app.event("app_mention")
def handle_mention(event, say):
    """处理@Bot的消息"""
    user = event["user"]
    text = event["text"]
    channel = event["channel"]
    
    # 处理消息
    response = process_message(text)
    say(text=response, channel=channel)

@app.event("message")
def handle_message(event, say):
    """处理普通消息"""
    # 忽略Bot自己的消息
    if event.get("bot_id"):
        return
    
    # 只处理私信
    if event.get("channel_type") == "im":
        text = event.get("text", "")
        response = process_message(text)
        say(text=response)

@app.command("/openclaw")
def handle_command(ack, command, say):
    """处理斜杠命令"""
    ack()
    text = command["text"]
    response = process_command(text)
    say(text=response)

if __name__ == "__main__":
    handler = SocketModeHandler(app, os.environ.get("SLACK_APP_TOKEN"))
    handler.start()
```

```python
def send_message(to, text, last_message_time):

    from datetime import datetime, timedelta
    
    # 检查是否在24小时会话窗口内
```
