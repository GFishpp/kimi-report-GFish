## 1. Browser 工具

### 1.1 功能概述

Browser 工具提供强大的浏览器控制能力，支持通过 Chrome 扩展中继或独立的 OpenClaw 管理浏览器进行网页自动化操作。主要功能包括：

- **浏览器控制**：启动、停止、管理浏览器实例
- **页面导航**：打开 URL、前进后退、刷新页面
- **元素交互**：点击、输入、悬停、选择等操作
- **页面分析**：获取 DOM 快照、截图、PDF 生成
- **多标签管理**：创建、切换、关闭标签页
- **JavaScript 执行**：在页面上下文中运行自定义脚本

### 1.2 参数详解

#### 核心参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `action` | string | 是 | 操作类型：status/start/stop/profiles/tabs/open/focus/close/snapshot/screenshot/navigate/console/pdf/upload/dialog/act |
| `profile` | string | 否 | 浏览器配置文件："chrome" 使用 Chrome 扩展中继，"openclaw" 使用独立浏览器 |
| `target` | string | 否 | 目标位置：sandbox/host/node |
| `targetId` | string | 否 | 目标标签页 ID |
| `targetUrl` | string | 否 | 目标 URL |
| `selector` | string | 否 | CSS 选择器 |
| `ref` | string | 否 | 元素引用 ID（从 snapshot 获取） |
| `refs` | string | 否 | 引用类型："role"（默认）或 "aria" |
| `fullPage` | boolean | 否 | 截图时是否捕获完整页面 |
| `type` | string | 否 | 截图格式：png/jpeg |
| `timeoutMs` | number | 否 | 操作超时时间（毫秒） |

#### Action 详细说明

| Action | 说明 | 常用参数 |
|--------|------|----------|
| `status` | 获取浏览器状态 | - |
| `start` | 启动浏览器 | profile, target |
| `stop` | 停止浏览器 | profile, target |
| `profiles` | 列出可用配置文件 | - |
| `tabs` | 列出所有标签页 | profile, target |
| `open` | 打开新标签页 | targetUrl, profile |
| `focus` | 聚焦到指定标签页 | targetId |
| `close` | 关闭标签页 | targetId |
| `navigate` | 导航到 URL | targetUrl, targetId |
| `snapshot` | 获取页面 DOM 快照 | targetId, refs, compact |
| `screenshot` | 页面截图 | targetId, fullPage, type |
| `pdf` | 生成 PDF | targetId |
| `console` | 获取控制台日志 | targetId, level |
| `upload` | 文件上传 | targetId, selector, paths |
| `dialog` | 处理对话框 | targetId, accept, promptText |
| `act` | 执行元素操作 | targetId, request |

#### Request 对象（用于 act action）

| 字段 | 类型 | 说明 |
|------|------|------|
| `kind` | string | 操作类型：click/type/press/hover/drag/select/fill/resize/wait/evaluate/close |
| `ref` | string | 元素引用 |
| `text` | string | 输入文本（type/fill 使用） |
| `key` | string | 按键（press 使用） |
| `button` | string | 鼠标按钮：left/right/middle |
| `doubleClick` | boolean | 是否双击 |
| `slowly` | boolean | 是否慢速操作 |
| `submit` | boolean | 是否提交表单 |
| `fn` | string | JavaScript 函数（evaluate 使用） |
| `timeMs` | number | 等待时间（wait 使用） |

### 1.3 使用示例

#### 示例 1：打开网页并获取快照

```json
{
  "action": "open",
  "profile": "openclaw",
  "targetUrl": "https://example.com"
}
```

获取快照：
```json
{
  "action": "snapshot",
  "targetId": "tab_xxx",
  "refs": "aria",
  "compact": true
}
```

#### 示例 2：点击元素并输入文本

```json
{
  "action": "act",
  "targetId": "tab_xxx",
  "request": {
    "kind": "click",
    "ref": "e12"
  }
}
```

输入文本：
```json
{
  "action": "act",
  "targetId": "tab_xxx",
  "request": {
    "kind": "type",
    "ref": "e15",
    "text": "搜索内容",
    "submit": true
  }
}
```

#### 示例 3：页面截图

```json
{
  "action": "screenshot",
  "targetId": "tab_xxx",
  "fullPage": true,
  "type": "png"
}
```

#### 示例 4：生成 PDF

```json
{
  "action": "pdf",
  "targetId": "tab_xxx"
}
```

#### 示例 5：执行 JavaScript

```json
{
  "action": "act",
  "targetId": "tab_xxx",
  "request": {
    "kind": "evaluate",
    "fn": "() => document.title"
  }
}
```

### 1.4 最佳实践

1. **使用 aria 引用**：设置 `refs: "aria"` 获得更稳定的元素引用
2. **合理设置超时**：复杂页面加载设置 `timeoutMs: 30000` 或更高
3. **复用 targetId**：从 snapshot 获取的 targetId 在后续操作中复用
4. **错误处理**：捕获超时和元素未找到异常
5. **资源清理**：使用完毕后关闭标签页或停止浏览器

### 1.5 常见错误处理

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| `No tab is connected` | Chrome 扩展未连接 | 点击浏览器工具栏的 OpenClaw 图标连接标签页 |
| `TimeoutError` | 页面加载超时 | 增加 `timeoutMs` 参数 |
| `Element not found` | 选择器或引用无效 | 重新获取 snapshot，检查元素是否存在 |
| `Browser not started` | 浏览器未启动 | 先执行 `action: "start"` |
| `Navigation failed` | 导航失败 | 检查 URL 和网络连接 |

## 2. Canvas 工具

### 2.1 功能概述

Canvas 工具用于控制节点画布，支持渲染 UI、执行 JavaScript、捕获快照等功能。主要用途包括：

- **画布展示**：在节点上展示可视化内容
- **A2UI 推送**：将 AI 生成的 UI 推送到画布
- **快照捕获**：获取画布当前状态的截图
- **JavaScript 执行**：在画布上下文中运行脚本
- **导航控制**：控制画布的页面导航

### 2.2 参数详解

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `action` | string | 是 | 操作类型：present/hide/navigate/eval/snapshot/a2ui_push/a2ui_reset |
| `node` | string | 否 | 目标节点 ID 或名称 |
| `target` | string | 否 | 目标标识 |
| `url` | string | 否 | 导航 URL（navigate 使用） |
| `javaScript` | string | 否 | 要执行的 JavaScript 代码（eval 使用） |
| `jsonl` | string | 否 | A2UI JSONL 数据（a2ui_push 使用） |
| `jsonlPath` | string | 否 | A2UI JSONL 文件路径 |
| `width` | number | 否 | 画布宽度 |
| `height` | number | 否 | 画布高度 |
| `x` | number | 否 | X 坐标位置 |
| `y` | number | 否 | Y 坐标位置 |
| `outputFormat` | string | 否 | 输出格式：png/jpg/jpeg |
| `quality` | number | 否 | JPEG 质量（0-100） |
| `delayMs` | number | 否 | 延迟时间（毫秒） |

#### Action 详细说明

| Action | 说明 |
|--------|------|
| `present` | 展示画布 |
| `hide` | 隐藏画布 |
| `navigate` | 导航到指定 URL |
| `eval` | 执行 JavaScript 代码 |
| `snapshot` | 捕获画布快照 |
| `a2ui_push` | 推送 A2UI 内容到画布 |
| `a2ui_reset` | 重置 A2UI 状态 |

### 2.3 使用示例

#### 示例 1：展示画布

```json
{
  "action": "present",
  "node": "node_001",
  "width": 1280,
  "height": 720
}
```

#### 示例 2：导航到 URL

```json
{
  "action": "navigate",
  "node": "node_001",
  "url": "https://example.com/dashboard"
}
```

#### 示例 3：执行 JavaScript

```json
{
  "action": "eval",
  "node": "node_001",
  "javaScript": "document.body.style.backgroundColor = 'blue'"
}
```

#### 示例 4：捕获快照

```json
{
  "action": "snapshot",
  "node": "node_001",
  "outputFormat": "png",
  "quality": 90
}
```

#### 示例 5：推送 A2UI 内容

```json
{
  "action": "a2ui_push",
  "node": "node_001",
  "jsonl": "{\"type\":\"component\",\"name\":\"Button\",\"props\":{\"label\":\"点击我\"}}"
}
```

### 2.4 最佳实践

1. **设置合适尺寸**：根据展示内容设置合理的 width/height
2. **延迟捕获**：截图前设置 `delayMs` 确保内容渲染完成
3. **节点指定**：明确指定 node 参数避免操作错误目标
4. **A2UI 格式**：确保 JSONL 格式正确，每行一个 JSON 对象

### 2.5 常见错误处理

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| `Canvas not found` | 画布未初始化 | 先执行 `action: "present"` |
| `Node not connected` | 节点未连接 | 检查节点 ID 和网络状态 |
| `JavaScript error` | JS 执行错误 | 检查代码语法和上下文 |
| `Invalid JSONL` | A2UI 数据格式错误 | 验证 JSONL 格式 |

## 3. Cron 工具

### 3.1 功能概述

Cron 工具用于管理定时任务，支持：

- **定时执行**：按指定时间间隔执行任务
- **调度语法**：支持标准 Cron 表达式
- **任务管理**：创建、列出、删除定时任务
- **子代理调度**：定时触发子代理执行特定任务

### 3.2 参数详解

> 注意：Cron 功能通常通过 `openclaw` CLI 命令使用，而非直接调用工具。

#### CLI 参数

| 参数 | 说明 |
|------|------|
| `schedule` | Cron 表达式（如 "0 9 * * *" 表示每天 9 点） |
| `command` | 要执行的命令 |
| `name` | 任务名称 |
| `enabled` | 是否启用 |

#### Cron 表达式格式

```
* * * * *
│ │ │ │ │
│ │ │ │ └── 星期几 (0-7, 0 和 7 都是周日)
│ │ │ └──── 月份 (1-12)
│ │ └────── 日期 (1-31)
│ └──────── 小时 (0-23)
└────────── 分钟 (0-59)
```

#### 特殊字符

| 字符 | 说明 |
|------|------|
| `*` | 任意值 |
| `,` | 列表分隔符（如 "1,3,5"） |
| `-` | 范围（如 "1-5"） |
| `/` | 步长（如 "*/5" 每 5 分钟） |

### 3.3 使用示例

#### 示例 1：每天上午 9 点执行任务

```bash
openclaw cron add --name "morning-report" --schedule "0 9 * * *" --command "openclaw run daily-report"
```

#### 示例 2：每 30 分钟执行一次

```bash
openclaw cron add --name "health-check" --schedule "*/30 * * * *" --command "openclaw run check-system"
```

#### 示例 3：每周一上午 8 点

```bash
openclaw cron add --name "weekly-sync" --schedule "0 8 * * 1" --command "openclaw run weekly-meeting"
```

#### 示例 4：列出所有定时任务

```bash
openclaw cron list
```

#### 示例 5：删除定时任务

```bash
openclaw cron remove --name "morning-report"
```

### 3.4 最佳实践

1. **命名规范**：使用描述性名称便于管理
2. **避免重叠**：确保任务执行时间不会重叠
3. **日志记录**：任务输出重定向到日志文件
4. **错误通知**：配置失败通知机制
5. **时区注意**：确认 Cron 使用系统时区

### 3.5 常见错误处理

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| `Invalid cron expression` | Cron 表达式格式错误 | 检查表达式语法 |
| `Command not found` | 命令不存在 | 使用绝对路径或检查 PATH |
| `Permission denied` | 权限不足 | 检查文件权限 |
| `Overlapping execution` | 任务执行时间过长 | 增加执行间隔或优化任务 |

## 4. Sessions 工具

### 4.1 功能概述

Sessions 工具用于管理执行会话，支持：

- **会话列表**：查看所有活动会话
- **会话控制**：发送输入、获取日志、终止会话
- **后台任务**：管理长时间运行的任务
- **子代理**：创建和管理子代理会话

### 4.2 参数详解

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `action` | string | 是 | 操作类型：list/poll/log/write/send-keys/submit/paste/kill |
| `sessionId` | string | 条件 | 会话 ID（除 list 外必填） |
| `data` | string | 否 | 写入的数据（write 使用） |
| `eof` | boolean | 否 | 写入后关闭 stdin |
| `keys` | array | 否 | 按键序列（send-keys 使用） |
| `hex` | array | 否 | 十六进制按键码 |
| `literal` | string | 否 | 字面量字符串 |
| `text` | string | 否 | 粘贴的文本 |
| `bracketed` | boolean | 否 | 使用括号模式粘贴 |
| `offset` | number | 否 | 日志偏移量 |
| `limit` | number | 否 | 日志行数限制 |

#### Action 详细说明

| Action | 说明 |
|--------|------|
| `list` | 列出所有会话 |
| `poll` | 轮询会话状态 |
| `log` | 获取会话日志 |
| `write` | 向会话写入数据 |
| `send-keys` | 发送按键序列 |
| `submit` | 提交输入 |
| `paste` | 粘贴文本 |
| `kill` | 终止会话 |

### 4.3 使用示例

#### 示例 1：列出所有会话

```json
{
  "action": "list"
}
```

#### 示例 2：获取会话日志

```json
{
  "action": "log",
  "sessionId": "session_xxx",
  "offset": 0,
  "limit": 100
}
```

#### 示例 3：向会话写入数据

```json
{
  "action": "write",
  "sessionId": "session_xxx",
  "data": "echo 'Hello World'",
  "eof": true
}
```

#### 示例 4：发送按键

```json
{
  "action": "send-keys",
  "sessionId": "session_xxx",
  "keys": ["Enter", "Ctrl+c"]
}
```

#### 示例 5：终止会话

```json
{
  "action": "kill",
  "sessionId": "session_xxx"
}
```

### 4.4 最佳实践

1. **及时清理**：用完会话后及时终止
2. **日志监控**：定期检查长时间运行会话的日志
3. **输入验证**：向会话发送数据前验证内容
4. **错误处理**：处理会话不存在或已结束的情况

### 4.5 常见错误处理

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| `Session not found` | 会话 ID 错误 | 检查 sessionId 是否正确 |
| `Session already ended` | 会话已结束 | 创建新会话 |
| `Write failed` | 写入失败 | 检查会话状态和权限 |

## 5. Nodes 工具

### 5.1 功能概述

Nodes 工具用于管理和控制配对的节点设备，支持：

- **节点发现**：发现和配对节点设备
- **设备控制**：相机、屏幕、位置等硬件控制
- **远程执行**：在节点上执行命令
- **通知推送**：向节点发送通知

### 5.2 参数详解

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `action` | string | 是 | 操作类型：status/describe/pending/approve/reject/notify/camera_snap/camera_list/camera_clip/screen_record/location_get/run/invoke |
| `node` | string | 否 | 节点 ID 或名称 |
| `deviceId` | string | 否 | 设备 ID |
| `title` | string | 否 | 通知标题 |
| `body` | string | 否 | 通知内容 |
| `sound` | string | 否 | 通知声音 |
| `priority` | string | 否 | 优先级：passive/active/timeSensitive |
| `delivery` | string | 否 | 投递方式：system/overlay/auto |
| `facing` | string | 否 | 相机朝向：front/back/both |
| `duration` | string | 否 | 录制时长 |
| `durationMs` | number | 否 | 录制时长（毫秒） |
| `quality` | number | 否 | 质量（0-100） |
| `maxWidth` | number | 否 | 最大宽度 |
| `command` | array | 否 | 要执行的命令（run 使用） |
| `cwd` | string | 否 | 工作目录 |
| `env` | array | 否 | 环境变量 |
| `timeoutMs` | number | 否 | 超时时间 |

#### Action 详细说明

| Action | 说明 |
|--------|------|
| `status` | 获取节点状态 |
| `describe` | 获取节点详细信息 |
| `pending` | 列出待配对请求 |
| `approve` | 批准配对请求 |
| `reject` | 拒绝配对请求 |
| `notify` | 发送通知 |
| `camera_snap` | 拍照 |
| `camera_list` | 列出相机 |
| `camera_clip` | 录制视频 |
| `screen_record` | 屏幕录制 |
| `location_get` | 获取位置 |
| `run` | 执行命令 |
| `invoke` | 调用节点方法 |

### 5.3 使用示例

#### 示例 1：列出所有节点

```json
{
  "action": "status"
}
```

#### 示例 2：发送通知

```json
{
  "action": "notify",
  "node": "iphone_001",
  "title": "提醒",
  "body": "会议即将开始",
  "priority": "active",
  "sound": "default"
}
```

#### 示例 3：拍照

```json
{
  "action": "camera_snap",
  "node": "iphone_001",
  "facing": "back",
  "quality": 90
}
```

#### 示例 4：录制屏幕

```json
{
  "action": "screen_record",
  "node": "iphone_001",
  "durationMs": 10000,
  "quality": 80,
  "includeAudio": true
}
```

#### 示例 5：获取位置

```json
{
  "action": "location_get",
  "node": "iphone_001",
  "desiredAccuracy": "precise",
  "timeoutMs": 30000
}
```

#### 示例 6：在节点执行命令

```json
{
  "action": "run",
  "node": "server_001",
  "command": ["ls", "-la"],
  "cwd": "/home/user",
  "timeoutMs": 60000
}
```

### 5.4 最佳实践

1. **权限管理**：确保节点已配对并授权
2. **超时设置**：长时间操作设置合理的 timeoutMs
3. **错误处理**：处理节点离线或无权限情况
4. **资源释放**：录制完成后及时保存文件
5. **隐私注意**：使用相机和位置功能时注意隐私合规

### 5.5 常见错误处理

| 错误 | 原因 | 解决方案 |
|------|------|----------|
| `Node not found` | 节点未配对 | 检查节点 ID 或重新配对 |
| `Permission denied` | 无权限 | 请求节点授权 |
| `Device offline` | 设备离线 | 检查网络连接 |
| `Camera not available` | 相机被占用 | 关闭其他使用相机的应用 |
| `Location unavailable` | 定位服务关闭 | 开启设备定位服务 |
| `Command failed` | 命令执行失败 | 检查命令语法和权限 |
